# What I'm building: (one sentence)
A web app that lets users browse and discover artists from the Art Institute
of Chicago's collection. Users will see a list of artists and be able to view details of each and explore their works in a gallery.

# Which API I'm using: (name and URL)
Art Institute of Chicago - api.artic.edu

# Why I chose this: (what interests you about this data?)
When I studied abroad in Paris, I really anjoyed all of the culturally rich museums. I have never been to Chicago, but am interested in the artwork kept at the Art Institute. This app will allow users to explore art and discover artists digitally.

# Core features: (3-5 things it will do)
- browse artists, see an individual's entire available work in gallery view
- learn about any selected artist's background information, such as nationality or significate dates
- loading state while API data is fetched
- error handling with an alert/message if the API fails or returns no results

# What I don't know yet: (what about fetch/async/JSON is new to you?)
- how the Art Institute structures their API
- how to link an artist to their artworks smoothly
- how to display the artwork images correctly
- best CSS practices for a responsive image gallery
